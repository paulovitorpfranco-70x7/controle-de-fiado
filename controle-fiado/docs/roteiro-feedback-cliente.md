# Roteiro de Feedback do Cliente

## Status

Piloto com cliente validado.

Resultado:

- dono validou o fluxo principal
- operacao com `OWNER` e `STAFF` aprovada
- fluxo de venda, pagamento e cobranca aprovado
- proxima etapa: ajustes finais de UI/visual

## Objetivo

Coletar feedback do dono e da operadora durante o piloto com dados reais,
sem misturar opiniao visual com erro operacional.

## Como conduzir

Use o sistema por alguns dias com poucos clientes reais e registre respostas
objetivas.

Evite perguntar apenas "gostou?". Prefira perguntas sobre operacao.

## Perguntas para o dono

1. Voce conseguiu consultar a lista de clientes pelo celular?
2. Voce conseguiu entender rapidamente quem esta devendo?
3. O saldo dos clientes ficou claro?
4. O pagamento parcial ficou claro?
5. A escolha de qual titulo quitar ajudou?
6. A cobranca pelo WhatsApp ficou facil de usar?
7. O texto da mensagem de cobranca esta adequado?
8. Faltou alguma informacao importante na ficha do cliente?
9. Alguma tela ficou confusa?
10. Voce confiaria em usar esse sistema no dia a dia?

## Perguntas para a operadora

1. Voce conseguiu cadastrar cliente sem dificuldade?
2. Voce conseguiu registrar venda rapidamente?
3. O campo de cliente na venda ficou claro?
4. O vencimento ficou facil de escolher?
5. Alguma informacao apareceu que voce nao deveria usar?
6. Algum texto ou botao ficou confuso?
7. O sistema ficou facil de usar no ritmo da loja?

## Pontos de observacao

Durante o piloto, observar:

- se a operadora demora para encontrar o cliente
- se o dono entende o saldo sem explicacao
- se pagamentos parciais geram duvida
- se a cobranca manual via WhatsApp e usada corretamente
- se algum telefone precisa ser corrigido manualmente
- se o uso no celular e confortavel

## Criterios para aprovar o piloto real

O piloto real pode ser aprovado quando:

- dono e operadora conseguem operar sem ajuda constante
- nenhum saldo fica incorreto
- nenhum vencimento aparece errado
- WhatsApp abre corretamente no celular do dono
- permissoes continuam coerentes
- o dono aprova seguir para uso real

## Criterios para pausar

Pausar e corrigir se:

- houver erro financeiro
- a operadora nao conseguir registrar venda com seguranca
- o dono nao entender saldo ou cobranca
- o WhatsApp falhar de forma recorrente
- houver confusao grave entre cliente, venda e pagamento

## Saida esperada

Ao final do piloto real, registrar:

- aprovado ou reprovado
- principais dificuldades
- ajustes obrigatorios
- ajustes desejaveis
- decisao sobre transformar o preview em ambiente definitivo
